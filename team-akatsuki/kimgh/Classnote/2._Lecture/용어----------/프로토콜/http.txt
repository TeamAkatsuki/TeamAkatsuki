===== HTTP (HyperText Transfer Protocol) =====

**웹에서 이루어지는 모든 데이터 교환의 기초, 개별적인 메시지 교환으로 통신**
**(contrast 데이터 스트림)**

__간단함__ - 사람이 읽을 수 있고, 간단한 고안, 테스트 쉽고, 초심자 진입장벽 낮춤

__확장 가능성__ - HTTP 헤더를 통해 확장, 실험 용이. 클라이언트, 서버가 새로운 헤더의
시맨틱 을 간단히 합의하면, 새로운 기능 추가 가능

__Stateless__ - 상태를 저장하지 않음. 
주로 HTML 문서를 주고 받는 데 쓰임
확장성 : 하이퍼텍스트 문서 -> 이미지, 비디오, HTML 폼 결과 포스팅

TCP, UDP 사용 +) TSL(암호화된 TCP)




==== 클라이언트-서버 프로토콜 ====

클라이언트와 서버 사이에 이루어지는 요청/응답(request/response) 프로토콜

ex) 웹 브라우저(클라이언트) - HTTP를 통해 서버에 정보 요청(request) - 서버 응답(response), 사용자에게 전달




**클라이언트:사용자 에이전트**


사용자를 대신하여 동작하는 모든 도구, 주로 브라우저. 브라우저는 항상 요청을 보내는 개체, 


- **클라이언트(브라우저)가 웹 페이지를 표시하기 위한 과정**

HTML 문서에 대한 요청을 전송
받은 HTML 파일을 구문 분석
실행해야 할 스크립트페이지 내 하위 리소스들
레이아웃 정보(CSS)에 대응하는 추가 요청을 받음

완전한 문서인 웹 페이지를 표시하기 위해 리소스들을 혼합
웹 페이지 갱신
웹 페이지 = 하이퍼텍스트 문서, 사용자가 사용자 에이전트를 제어하고 웹서핑을 할 수 있도록, 실행될 수 있는 링크임을 의미.)

**프록시**

브라우저와 서버 사이에서 수많은 컴퓨터, 머신 존재. 상호 HTTP 메시지 이어 받고 전달.
여러 계층 사이에서 보통 전송, 네트워크, 물리 계층에서 동작하며 큰 영향력을 행사하지만 
HTTP 계층에서는 동작이 보이지 않음. 

이 중에 애플리케이션 계층에서 동작하는 것들을 일반적으로 프록시라고 부름.
프록시는 눈에 보이거나, 그렇지 않을 수도 있음.

	캐싱 - 공개, 혹은 비공개 캐시 ex) 브라우저 캐시
	필터링 - 바이러스 백신 스캔, 컨텐츠 차단 기능
	로드 밸런싱 - 여러 서버들이 서로 다른 요청 처리하도록 허용
	인증 - 다양한 리소스에 대한 접근 제어
	로깅 - 이력 정보를 저장



**웹 서버**

클라이언트에 의한 요청에 대한 문서를 제공, 논리적으로는 단일 기계이지만 반드시 단일 머신일 필요는 없음.
